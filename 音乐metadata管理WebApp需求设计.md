# 音乐 Metadata 管理 Web App — 需求与功能设计

> 个人本地音乐库管理工具，支持 FLAC / AAC (M4A) 格式
> 技术栈：FastAPI (Python) · Vue 3 · uv 包管理

---

## 一、产品定位

运行在本地的轻量 Web 应用，让你用浏览器界面管理本机硬盘上的音乐文件 metadata。**所有操作发生在同一个页面内，无子页面、无路由跳转**，像桌面软件一样直接。

---

## 二、整体布局（单页面）

### 核心设计原则

**右栏永远是一个卡片列表**，没有"单选模式"和"多选模式"之分：
- 选中 1 首 → 右栏显示 1 张卡片
- 选中 N 首 → 右栏显示 N 张卡片，竖向堆叠，上下滚动
- 未选中任何歌曲 → 右栏显示空状态提示

这让右栏的渲染逻辑完全统一，无需两套 UI。

```
┌──────────────────────────────────────────────────────────────────┐
│  顶栏：[📁 文件夹路径输入框                          ] [打开] [⚙] │
├───────────────────────┬──────────────────────────────────────────┤
│  左栏（固定 300px）    │  右栏（卡片列表，独立滚动）                │

│                       │  ┌── 歌曲C.m4a ───────────────────────┐  │

**左栏选中行为：**
- 点击歌曲行的**非 checkbox 区域** → 清除所有勾选，仅选中该首歌（右栏变为单张卡片）
- 点击歌曲行的 **checkbox** → 追加/取消勾选，右栏增减对应卡片
- 点击「全选」checkbox → 勾选当前过滤列表中的所有歌曲

**右栏工具栏（固定悬浮于卡片区上方）：**
- 始终显示当前卡片数量（即已选歌曲数）
- 所有按钮**始终针对右栏当前显示的所有卡片**执行，无论有 1 张还是 50 张
- 「全部保存」：保存所有有改动的卡片，无改动的跳过
- 「搜索封面」：为所有卡片的歌曲逐一搜索并填入封面
- 「获取歌词」：为所有卡片的歌曲逐一下载 LRC
- 「同步专辑封面」：在当前卡片范围内，按专辑分组同步封面
- 按钮**始终可用**，不因卡片数量而灰显

**卡片内容：** 每张卡片结构完全一致——封面、所有 metadata 字段、封面操作按钮、LRC 区块、单独保存按钮。未保存时卡片顶部边框变色提示。

## 三、功能模块详细设计

### 1. 打开文件夹

- 顶栏有一个路径输入框 +「打开」按钮
- 点击「打开」→ 后端**递归扫描**该目录及所有子目录下的 `.flac` / `.m4a` / `.aac` 文件（适配 Artist/Album/song.flac 层级结构）
- 扫描完成后左栏列表立刻刷新，保留上次路径（写入 `localStorage`）
- 支持 `~` 路径展开（如 `~/Music`）
- 扫描中显示 loading 状态，扫描完成显示歌曲总数

### 2. 歌曲列表（左栏）

#### 2.1 视图模式（可切换）

左栏顶部有**视图切换下拉**，两种模式：

**平铺模式（默认）**
- 所有歌曲在同一个滚动列表中展开
- 每条目：checkbox + 歌曲标题（主）+ 艺术家（次）+ LRC 徽标（若有）
- 排序下拉可选：按艺术家、按专辑、按标题、按文件名

**树形模式**
- 按 艺术家 → 专辑 两级分组，可折叠/展开
- 艺术家节点显示名称 + 歌曲总数
- 专辑节点显示专辑名 + 曲目数
- 叶节点为单首歌曲，带 checkbox 和 LRC 徽标
- 折叠/展开状态保存在 `localStorage`

#### 2.2 多选机制

- 左栏顶部有「全选 / 全不选」checkbox
- 每条歌曲左侧有 checkbox，可单独勾选
- 树形模式下，勾选专辑节点 = 勾选该专辑所有歌曲
- 选中 ≥ 1 首时，右栏自动切换为批量操作面板
- 取消所有勾选后，右栏恢复到上次单选的歌曲编辑器

#### 2.3 列表信息密度（可配置）

设置里可以控制两个维度：

**显示模式**
- **详细模式**：每条目较高，可显示更多字段
- **简略模式**：每条目紧凑，仅显示核心信息，适合大型音乐库快速浏览

**可开关的列（在设置中逐项勾选）**

| 列 | 说明 | 默认 |
|----|------|------|
| 封面缩略图 | 小方块封面图（~36px），无封面显示灰色占位 | 关 |
| 歌曲标题 | 主标识，始终显示，不可关闭 | 开 |
| 艺术家 | 副标识 | 开 |
| 专辑名 | 第三行或内联字段 | 关 |
| 时长 | 从音频文件读取，格式 `m:ss` | 关 |
| 格式徽标 | `FLAC` / `AAC` 彩色小标签 | 开 |
| LRC 徽标 | 蓝色 `LRC` 标签，无歌词则灰色占位或不显示 | 开 |

扫描时一并读取时长（mutagen 提供 `info.length`），存入内存索引，无需额外读取文件。封面缩略图开启时首次扫描会慢一些（需读取所有封面数据），扫描完成后缓存在内存中。

#### 2.4 条目其他交互

- 当前单选的歌曲：行背景高亮
- 支持键盘 ↑↓ 方向键在平铺模式下切换选中歌曲
- 树形模式下折叠/展开状态保存到 `localStorage`

### 3. 搜索

- 左栏顶部有搜索框，输入即时过滤（无需回车）
- 匹配范围：**歌曲标题 / 艺术家 / 专辑名**，不区分大小写
- 搜索结果为空时显示 "无匹配结果"
- 清空搜索框恢复全部列表

### 4. Metadata 编辑（右栏）

所有字段直接显示为输入控件，无额外点击：

| 字段 | 控件类型 | FLAC 标签 | M4A 标签 |
|------|---------|-----------|----------|
| 标题 | 单行文本 | `TITLE` | `©nam` |
| 艺术家 | 单行文本 | `ARTIST` | `©ART` |
| 专辑艺术家 | 单行文本 | `ALBUMARTIST` | `aART` |
| 专辑 | 单行文本 | `ALBUM` | `©alb` |
| 年份 | 4位数字 | `DATE` | `©day` |
| 曲目编号 | `数字/总数` | `TRACKNUMBER` | `trkn` |
| 碟号 | 数字 | `DISCNUMBER` | `disk` |
| 流派 | 单行文本 | `GENRE` | `©gen` |
| 作曲 | 单行文本 | `COMPOSER` | `©wrt` |
| 评论 | 多行文本 | `COMMENT` | `©cmt` |

- 任何字段改动后，「保存到文件」按钮变为高亮状态
- 点击保存 → 后端用 mutagen 直接写回原文件
- 保存成功后显示短暂的 ✓ 提示

### 5. 封面管理

封面区域显示当前封面大图（约 180×180px），无封面时显示占位图。

#### 5.1 本地上传
- 点击封面图片区域 或 拖拽图片文件 → 选择本地 JPG/PNG 文件
- 选择后立即在右栏预览，标记为待保存状态

#### 5.2 在线搜索封面（iTunes + 网易云）
- 点击「搜索封面」按钮 → 弹出搜索面板（覆盖层，非新页面）
- 以当前歌曲的 **艺术家 + 专辑名** 为关键词，**同时**查询两个来源：
  - **iTunes Search API**：欧美音乐覆盖好，图片清晰
  - **网易云音乐（pyncm）**：华语/日韩音乐匹配更准
- 两个来源的结果合并显示，每个候选显示缩略图 + 专辑名 + 来源标签
- 展示最多 10 个候选（两个来源各取前 5）
- 点击某个封面 → 后端下载 → 预览显示在右栏，标记为待保存

#### 5.3 从库中其他歌曲复制封面
- 点击「从库复制」→ 弹出选择面板
- 列出当前库中所有**有封面**的歌曲（小缩略图 + 标题 + 艺术家）
- 支持在此面板内搜索
- 点击选中某首歌 → 提取其封面 → 预览显示在右栏，标记为待保存

#### 5.4 一键同步封面到整张专辑（核心功能）
- 按钮：**「⚡ 同步封面到整张专辑」**
- 若当前有未保存的封面改动，先提示"建议先保存当前封面"
- 点击后弹确认框，列出将被修改的所有歌曲文件名及数量
  - 条件：专辑名与当前歌曲相同（不区分大小写）
- 用户确认后后端批量写入，完成后显示 "已更新 X 首歌曲"
- 左栏列表对应歌曲的封面图标同步刷新

### 6. 工具栏操作（作用于右栏所有卡片）

工具栏固定悬浮于卡片区上方，所有操作的**目标**始终是右栏当前显示的全部卡片（即左栏当前选中的所有歌曲）。

```
┌── 工具栏（固定悬浮）──────────────────────────────────────────┐
│  共 12 首        [💾 全部保存]                                │
│  [🖼 搜索封面]  [⚡ 同步专辑封面]  [🎵 获取歌词]              │
│                                                             │
│  （操作进行中时显示进度）                                      │
│  [████████████░░░░] 9 / 12   正在处理：青花瓷.m4a            │
└─────────────────────────────────────────────────────────────┘
```

#### 搜索封面（工具栏）
- 对右栏每张卡片的歌曲，以 `艺术家 + 专辑名` 同时查询 iTunes + 网易云
- **唯一结果** → 填入该卡片封面预览，标记为待保存（不立即写文件，需点各卡片「保存」或「全部保存」）
- **多个结果 / 无结果** → 跳过，计入完成报告
- 已有封面的歌曲**跳过**（不覆盖），可在执行前勾选「覆盖已有封面」

#### 获取歌词（工具栏）
- 对右栏每张卡片逐一请求 LRCLIB + pyncm
- **唯一结果** → 直接写入 `.lrc` / `.trans.lrc`（歌词立即落盘，不需要额外保存）
- **多个结果 / 无结果** → 跳过，计入报告
- 已有 LRC 的歌曲**跳过**（不覆盖）

#### 同步专辑封面（工具栏）
- 在右栏所有卡片中，按专辑名分组
- 每组内找到有封面的卡片，将其封面写入同组内无封面的歌曲
- 若某张专辑的所有卡片都无封面，在报告中提示

#### 操作完成报告
操作结束后，工具栏下方出现结果摘要（可收起）：

```
封面搜索完成：已填入 8 首  跳过 4 首（需手动处理）
  · 青花瓷.m4a          → 多个候选
  · 星晴.flac           → 未找到
  · Bohemian Rhapsody   → 多个候选
```

- 点击报告中的**文件名** → 左栏清除多选、单选跳转到该歌曲，右栏变为该曲单卡片，方便手动处理

### 7. 设置页（⚙）

顶栏右侧齿轮按钮打开设置面板（覆盖层）：

| 设置项 | 说明 |
|--------|------|
| 网易云 Cookie | 粘贴从浏览器复制的 Cookie 字符串，用于 `pyncm` 鉴权，解锁完整歌词和封面库 |
| 默认扫描路径 | 记住上次打开的文件夹路径 |
| 封面压缩质量 | 最大边长（默认 1500px）和 JPEG 品质（默认 90） |
| LRC 来源优先级 | 网易云优先 / LRCLIB 优先（影响唯一结果自动保存时选哪个） |
| 列表显示模式 | 详细 / 简略 |
| 列表显示列 | 多选：封面缩略图、艺术家、专辑、时长、格式徽标、LRC 徽标 |

**Cookie 填写说明**（展示在设置面板内）：
> 打开浏览器 → 登录 music.163.com → F12 → Network → 任意请求 → 复制 Request Headers 中的 `Cookie` 字段值粘贴至此

### 8. LRC 歌词管理

#### 6.1 名词定义

- **原文**：歌曲原始语言的歌词（日语歌就是日语，英语歌就是英语）
- **官方**：带时间轴的同步 LRC（非歌迷手打，来自官方授权或平台结构化数据库）
- **翻译**：中文译文歌词（通常来自网易云的 `tlyric` 字段）

#### 6.2 LRC 文件约定

- LRC 文件与音频文件**同目录、同名**，后缀改为 `.lrc`
  - 例：`青花瓷.m4a` → `青花瓷.lrc`（存原文歌词）
- 中文翻译存为 `.trans.lrc`
  - 例：`青花瓷.trans.lrc`（存译文，若有）
- 递归扫描后 LRC 状态也一并检测，LRC 文件必须与音频**同目录**

#### 6.3 LRC 状态显示

- **左栏歌曲列表**：每条目右侧显示 `LRC` 徽标（有原文歌词）或灰色占位（无歌词）
- **右栏编辑区**：在 metadata 字段下方有独立的「歌词」区块，显示：
  - 原文 LRC：✓ 已有 / ✗ 无
  - 中文译文：✓ 已有 / ✗ 无（仅在有 `.trans.lrc` 时显示 ✓）

#### 6.4 LRC 获取逻辑

歌词区块有**「获取歌词」**按钮，点击后：

1. 以当前歌曲的 **艺术家 + 标题（+ 专辑）** 为关键词，**同时**请求两个来源
2. 合并结果，根据匹配数量决定行为：
   - **唯一结果** → 直接静默保存，无需确认，完成后显示 ✓ 提示
   - **多个结果** → 弹出选择面板让用户挑选

**歌词来源（同时查询，均接受）：**

| 来源 | 提供内容 | 特点 |
|------|---------|------|
| **LRCLIB** (lrclib.net) | 原文 synced LRC | 免费开放 API，无需鉴权，欧美音乐覆盖好 |
| **网易云音乐**（非官方逆向 API） | 原文 LRC + 中文译文 LRC | 华语/日韩/欧美覆盖最全，接受其不稳定风险 |

- 结果按匹配度排序，同来源优先展示有时间轴的版本（synced 优于 plain text）
- 网易云返回的 `tlyric`（译文）同步存为 `.trans.lrc`

#### 6.5 多结果选择面板

```
┌── 获取歌词：周杰伦 - 青花瓷 ──────────────────────────────────┐
│  搜索词（可修改）：[周杰伦        ] [青花瓷              ] [搜索] │
│  ──────────────────────────────────────────────────────────  │
│  ● 来源：网易云  ✦ 带时间轴  ✦ 含中文译文                        │
│    [00:12.34] 素胚勾勒出青花笔锋浓转淡                           │
│    [00:15.89] 瓶身描绘的牡丹一如你初妆  ...  [展开全部 ▾]        │
│                                                              │
│  ○ 来源：LRCLIB  ✦ 带时间轴  （无译文）                          │
│    [00:12.50] 素胚勾勒出青花笔锋浓转淡  ...  [展开全部 ▾]        │
│  ──────────────────────────────────────────────────────────  │
│  保存项：[✓ 原文 .lrc]  [✓ 中文译文 .trans.lrc]                 │
│                                              [取消]  [保存]   │
└────────────────────────────────────────────────────────────┘
```

- 每个候选显示：来源、是否有时间轴、是否含译文、前几行预览
- 用户选择一个候选后，可分别勾选保存原文 / 译文
- 若已有 LRC 文件，面板顶部提示「将覆盖已有歌词」

#### 6.6 LRC 查看与手动编辑

- 已有 LRC 的歌曲，歌词区块展示前 5 行预览（点击可展开全文）
- 「编辑」按钮：弹出 `<textarea>` 可手动修改 LRC 内容，保存写回文件
- 「删除」按钮：可分别删除原文 `.lrc` 或译文 `.trans.lrc`

## 四、技术架构

### 4.1 整体结构

```
music-tag-app/
├── pyproject.toml          # uv 管理，声明依赖和项目元信息
├── uv.lock                 # 锁定依赖版本（由 uv 自动生成）
├── settings.json           # 运行时配置（Cookie、压缩参数等），gitignore
├── main.py                 # FastAPI 入口，同时 serve 前端静态文件
├── routers/
│   ├── library.py          # 扫描目录、列出歌曲
│   ├── songs.py            # 单曲 metadata 读写
│   ├── cover.py            # 封面相关（上传、iTunes、复制、同步专辑）

## 五、关键交互细节

### 切换歌曲时的未保存处理
- 若右栏有未保存修改，点击左栏其他歌曲时弹原生 `confirm()` 对话框
- 确认放弃 → 切换；取消 → 停留当前歌曲

### 封面图片处理规范
- 写入前统一压缩：最大边 1500px，JPEG 品质 90%（由后端 Pillow 处理）
- M4A 封面统一存为 JPEG；FLAC 封面保留原格式（JPEG/PNG）
- 封面大小超过 500KB 时自动降质

### 在线封面搜索流程（iTunes + 网易云）
1. 前端点击「搜索封面」，读取当前 `artist` / `album` 字段值作为搜索词（可手动修改）
2. 后端**并行**请求两个来源：
   - iTunes：`https://itunes.apple.com/search?term=...&entity=album&limit=5`，`artworkUrl100` → `artworkUrl600`
   - 网易云（pyncm）：搜索专辑，取封面图 URL
3. 合并两个来源最多 10 个候选，每个标注来源
4. 用户点选 → 后端 `POST /api/cover/download` 下载图片 → 返回 base64 → 前端设为待保存封面

### 同步专辑封面的匹配规则
- 以 **专辑名（去首尾空格，不区分大小写）** 作为唯一匹配依据
- 范围限定在**当前已扫描的文件夹**内
- 已有相同封面的歌曲也会被覆盖写入（幂等操作）
